# Prototypes

Two kinds of artifact live here. **Prototypes** are throwaway HTML design
artifacts — **not product code**: one self-contained static `.html` file each,
served verbatim by the spec site and linked (or iframed) from the spec pages it
illustrates. Start a new one by copying
[`prototype-skeleton.html`](prototype-skeleton.html). **Signed-off captures**
are durable: when the human approves a design, its screenshot(s) land in
`assets/` with a `<slug>.md` page here embedding them — the record of exactly
what was approved.
